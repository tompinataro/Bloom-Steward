Bloom Steward Full-Stack Request Flow

Illustrating how user and steward/admin requests flow through the system, with integrated modules and vendor technologies.

💻

1. Client (Front-end)

Presentation Layer

Built with React/Next.js. Includes ASC (App Steward Console) for steward/admin access.

→
🌐

2. API Interface

Transport Layer

Implemented via FastAPI / GraphQL Gateway. Steward routes enriched with EAS (Enhanced Admin Suite).

→
⚙️

3. Application Server

Logic Layer

Business logic in Python (Django/Flask). Extended by:

  • Automation Jobs (Apache Airflow / Celery)
  • Telemetry (Prometheus, Grafana)
  • Governance Rules (custom steward policies)
→
⇄

4. Data Access Layer

Translation Layer

ORM-based access via SQLAlchemy/Django ORM. Stewardship adds Data Governance Checks before DB calls.

→
💾

5. Database

Storage Layer

Primary datastore: PostgreSQL. Logs and steward data persisted in Elastic / Stewardship DB.

The Return Journey (Response)

Results are formatted as JSON/GraphQL responses, returned through the API, and rendered in the client (with ASC views for stewards).
